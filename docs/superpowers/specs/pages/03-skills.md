### コンポーネント詳細

#### 1. KPIカード (`components/skills/skill-kpi-cards.tsx`)

| カード | 値 | 説明 |
|---|---|---|
| 配布スキル数 | `distributed_skills`テーブルの登録数 | 管理対象のスキル総数 |
| 活用率 | 期間内に1回以上使用された配布スキル / 配布スキル総数 × 100 | チーム全体での浸透度 |
| 未活用スキル数 | 期間内に誰にも使用されなかった配布スキル数 | 改善が必要なスキル数 |

#### 2. スキル x メンバー ヒートマップ (`components/skills/skill-heatmap.tsx`)

配布スキルごとに各メンバーの使用頻度をヒートマップで表示する。

- Y軸: 配布スキル名
- X軸: メンバー名（アバター付き）
- セルの色: 使用回数に応じた濃淡（0回=白/薄灰、多い=濃い青）
- セルホバー: スキル名、メンバー名、使用回数をツールチップ表示
- Y軸のスキル名はクリックでスキルトレンドチャートをそのスキルにフォーカス

#### レンダリング

- メンバー数が多い場合（10人超）は横スクロールで対応
- スキル数が多い場合（20件超）はページ内スクロール（最大高さ制限）

#### データ取得クエリ

```sql
SELECT
  ds.skill_name,
  u.id AS user_id,
  u.name AS user_name,
  u.avatar_url,
  COUNT(su.id) AS usage_count
FROM distributed_skills ds
CROSS JOIN users u
LEFT JOIN sessions s ON s.user_id = u.id AND s.started_at BETWEEN :from AND :to
LEFT JOIN skill_usages su ON su.session_id = s.id AND su.skill_name = ds.skill_name
GROUP BY ds.skill_name, u.id, u.name, u.avatar_url
ORDER BY ds.skill_name, u.name;
```

#### 3. 未活用スキル一覧 (`components/skills/unused-skills.tsx`)

期間内に誰にも使用されなかった配布スキルを一覧表示する。

```
┌─────────────────────────────────────┐
│  未活用スキル                        │
│                                     │
│  ⚠ brainstorming                    │
│    登録日: 2026-03-15               │
│    説明: アイデアをデザインに変換      │
│                                     │
│  ⚠ code-reviewer                    │
│    登録日: 2026-03-20               │
│    説明: コードレビューの自動化       │
└─────────────────────────────────────┘
```

- 各スキルに警告アイコンと登録日、説明を表示
- スキルが0件の場合は「全スキルが活用されています」メッセージ

#### データ取得クエリ

```sql
SELECT ds.skill_name, ds.description, ds.registered_at
FROM distributed_skills ds
WHERE ds.skill_name NOT IN (
  SELECT DISTINCT su.skill_name
  FROM skill_usages su
  JOIN sessions s ON s.id = su.session_id
  WHERE s.started_at BETWEEN :from AND :to
)
ORDER BY ds.registered_at;
```

#### 4. スキルトレンド (`components/skills/skill-trend-chart.tsx`)

選択されたスキル（またはTop5スキル）の日別使用回数を折れ線グラフで表示する。

- 複数スキルを色分けして同時表示（最大5本）
- デフォルト: 使用回数Top5のスキルを表示
- ヒートマップのスキルをクリックすると、そのスキルにフォーカス
- 凡例でスキルの表示/非表示を切り替え可能

#### データ取得クエリ

```sql
SELECT
  DATE(su.invoked_at) AS date,
  su.skill_name,
  COUNT(*) AS usage_count
FROM skill_usages su
JOIN sessions s ON s.id = su.session_id
JOIN distributed_skills ds ON ds.skill_name = su.skill_name
WHERE s.started_at BETWEEN :from AND :to
GROUP BY DATE(su.invoked_at), su.skill_name
ORDER BY date, su.skill_name;
```

## スキル管理タブ

### レイアウト

```
┌─────────────────────────────────────────────────────────┐
│  配布スキル管理                    [+ スキルを追加]       │
├─────────────────────────────────────────────────────────┤
│ ┌──────────────┬─────────────────┬──────────┬─────────┐ │
│ │ スキル名      │ 説明            │ 登録日    │ 操作    │ │
│ ├──────────────┼─────────────────┼──────────┼─────────┤ │
│ │ brainstorming│ アイデアを...    │ 03/15    │ [編集]  │ │
│ │              │                 │          │ [削除]  │ │
│ │ debugging    │ バグの調査...    │ 03/15    │ [編集]  │ │
│ │              │                 │          │ [削除]  │ │
│ │ tdd          │ テスト駆動...    │ 03/20    │ [編集]  │ │
│ │              │                 │          │ [削除]  │ │
│ └──────────────┴─────────────────┴──────────┴─────────┘ │
└─────────────────────────────────────────────────────────┘
```

### 操作

#### スキル追加ダイアログ

```
┌────────────────────────────────┐
│  スキルを追加                   │
│                                │
│  スキル名 *                     │
│  [_________________________]   │
│                                │
│  説明                          │
│  [_________________________]   │
│  [_________________________]   │
│                                │
│  [キャンセル]  [追加]           │
└────────────────────────────────┘
```

- スキル名は必須、一意制約あり
- 説明はオプション
- 追加後はテーブルに即反映

#### スキル編集

- 編集ボタンでインラインまたはダイアログで編集
- スキル名と説明を変更可能
- スキル名を変更した場合、既存の`skill_usages`との紐付けは切れる旨を警告表示

#### スキル削除

- 削除ボタンで確認ダイアログを表示
- 「このスキルを削除すると、活用分析の対象から外れます。使用履歴は保持されます。」
- Server Actionで`distributed_skills`テーブルから削除（`skill_usages`は削除しない）

## Server Component / Client Component 分割

| コンポーネント | 種別 | 理由 |
|---|---|---|
| `SkillsPage` | Server | データ取得 |
| `SkillKpiCards` | Server | 静的表示 |
| `SkillHeatmap` | Client | インタラクティブなヒートマップ |
| `UnusedSkills` | Server | 静的リスト表示 |
| `SkillTrendChart` | Client | Recharts + スキル選択インタラクション |
| `SkillManagement` | Client | CRUD操作のインタラクション |

## ローディング・エラー状態

| 状態 | 表示 |
|---|---|
| ローディング | ヒートマップ: グリッドスケルトン、チャート: チャートスケルトン |
| 配布スキル未登録 | 「配布スキルが登録されていません。スキル管理タブから追加してください。」 |
| データなし | 「選択期間にスキル使用データがありません」 |
| エラー | 「データの取得に失敗しました」+ リトライボタン |
