# Members ページ仕様書

**パス:** `/members` (一覧) / `/members/[userId]` (詳細)
**目的:** 各メンバーのClaude Code使用状況を詳細に把握し、個人の改善に繋げる

---

### テーブルカラム

| カラム | 内容 | ソート |
|---|---|---|
| アバター | GitHubアバター画像 | - |
| メンバー | 名前（クリックで詳細ページへ遷移） | 名前昇順/降順 |
| セッション数 | 期間内のセッション数 | 数値ソート |
| トークン消費量 | 期間内の総トークン数（K/M単位で丸め表示） | 数値ソート |
| スキル活用率 | 配布スキルの使用率（%） | 数値ソート |
| 最終活動 | 最後のセッション開始からの経過時間 | 日時ソート |

### インタラクション

- テーブルヘッダーをクリックでソート切り替え（デフォルト: トークン消費量の降順）
- 検索バーでメンバー名を部分一致フィルタ（デバウンス300ms）
- 行クリックで `/members/[userId]` に遷移
- ページネーション: 1ページあたり20件

### データ取得クエリ

```sql
SELECT
  u.id,
  u.name,
  u.avatar_url,
  COUNT(DISTINCT s.id) AS session_count,
  COALESCE(SUM(tu.input_tokens + tu.output_tokens), 0) AS total_tokens,
  COUNT(DISTINCT CASE WHEN ds.id IS NOT NULL THEN su.skill_name END) AS used_distributed_skills,
  (SELECT COUNT(*) FROM distributed_skills) AS total_distributed_skills,
  MAX(s.started_at) AS last_activity
FROM users u
LEFT JOIN sessions s ON s.user_id = u.id AND s.started_at BETWEEN :from AND :to
LEFT JOIN token_usages tu ON tu.session_id = s.id
LEFT JOIN skill_usages su ON su.session_id = s.id
LEFT JOIN distributed_skills ds ON ds.skill_name = su.skill_name
GROUP BY u.id, u.name, u.avatar_url
ORDER BY total_tokens DESC
LIMIT :limit OFFSET :offset;
```

### コンポーネント詳細

#### 1. メンバーヘッダー (`components/members/member-header.tsx`)

- アバター画像（大）、名前、メールアドレス
- 「← 戻る」リンクで `/members` に遷移

#### 2. KPIカード (`components/members/member-kpi-cards.tsx`)

| カード | 値 | 前期間比 |
|---|---|---|
| セッション数 | 個人の期間内セッション数 | 前期間比 |
| トークン消費量 | 個人の期間内トークン数 | 前期間比 |
| サブエージェント使用数 | 個人の期間内サブエージェント呼び出し数 | 前期間比 |
| スキル活用率 | 配布スキル使用率（%） | 前期間比 |

#### 3. 使用推移チャート (`components/members/usage-trend-chart.tsx`)

日別の使用推移を折れ線グラフで表示する。タブで表示メトリクスを切り替え。

| タブ | Y軸 | 説明 |
|---|---|---|
| セッション数 | セッション開始数 | 日別のセッション回数 |
| トークン量 | トークン数（千単位） | 日別のトークン消費量 |
| スキル数 | スキル呼び出し数 | 日別のスキル使用回数 |

#### 4. スキル使用ランキング (`components/members/skill-ranking.tsx`)

個人が期間内に使用したスキルのランキング。

- 横棒グラフ（最大10件）
- 配布スキルは青色、未登録スキルはグレー
- 使用回数を表示

#### 5. プロジェクト別内訳 (`components/members/project-breakdown.tsx`)

セッションのprojectフィールドでグループ化し、プロジェクト別のセッション数をドーナツチャートで表示。

- 最大8プロジェクト表示、それ以上は「その他」にまとめる
- ホバーでプロジェクト名とセッション数を表示

#### 6. サブエージェント使用内訳 (`components/members/subagent-chart.tsx`)

日別のサブエージェント使用を、エージェントタイプ別に積み上げ棒グラフで表示。

| エージェントタイプ | 色 |
|---|---|
| Explore | 青 |
| Plan | 緑 |
| code-reviewer | 紫 |
| general-purpose | オレンジ |
| その他 | グレー |

#### 7. 最近のセッション一覧 (`components/members/recent-sessions.tsx`)

| カラム | 内容 |
|---|---|
| 開始日時 | セッションの開始時刻 |
| プロジェクト | 作業ディレクトリ名 |
| ブランチ | 作業ブランチ名（なしの場合は「-」） |
| 時間 | セッション時間（started_at 〜 ended_at） |
| トークン | セッション内のトークン消費量 |
| スキル | セッション内で使用したスキル名をバッジ表示 |
| サブエージェント | 使用したサブエージェントタイプをバッジ表示 |

- デフォルト: 最新20件表示
- 「もっと見る」ボタンで追加読み込み（無限スクロールではなくボタン方式）
- ブランチ名がPRに紐づいている場合はPRへのリンクアイコンを表示

## Server Component / Client Component 分割

| コンポーネント | 種別 | 理由 |
|---|---|---|
| `MembersListPage` | Server | テーブルデータのサーバー取得 |
| `MemberDetailPage` | Server | 個人データのサーバー取得 |
| `MemberKpiCards` | Server | 静的表示 |
| `UsageTrendChart` | Client | Recharts + タブ切り替え |
| `SkillRanking` | Client | Recharts |
| `ProjectBreakdown` | Client | Recharts |
| `SubagentChart` | Client | Recharts |
| `RecentSessions` | Client | 「もっと見る」の動的読み込み |

## ローディング・エラー状態

| 状態 | 表示 |
|---|---|
| ローディング | テーブル: 行スケルトン、チャート: チャートスケルトン |
| データなし | 一覧: 「メンバーが登録されていません」、詳細: 「選択期間にデータがありません」 |
| ユーザー不存在 | `/members/[存在しないID]` → `notFound()` で404表示 |
| エラー | 「データの取得に失敗しました」+ リトライボタン |
